Careless Safety Spectrum

  • 5Carefree Right action is easy. Wrong action is structurally hard.
  • 4Prevention Mistakes can't propagate past the originator
  • 3Deterministic Known classes caught reliably; predictable gaps
  • 2Probabilistic Sometimes caught; unpredictable gaps
  • 1Vigilance Caught only when someone is paying attention
  • 0Hope No mechanism. Errors propagate undetected.

The model names them

Worries

The gut-check developers feel about a kind of work.
Named for what experienced developers recognize.

Capability regression  ·  Accidental behavior change  ·  Data loss
Scope enforcement gap  ·  Decision inconsistency  ·  Oversight mechanism gap

Not abstract risk — the specific thing you're being careful about, every time.

The Recipe

  1. 1Identify an important kind of work
  2. 2Name the most impactful vigilance cost we're paying
  3. 3Look at the Worry — what safety level are we at?
  4. 4Look at safety options that could reach level 4 or 5
  5. 5Craft the AI's universe to implement the chosen option

The Worry:
Decision inconsistency

"Does this contradict a decision from earlier in the session?
Are we building on conflicting assumptions?"

Universe: Memory

  • Delete completed stories entirely — don't mark done, delete
  • Rewrite all future stories to reflect the current direction
  • Reset context to the base state

The AI cannot blend old ideas with new ones.
It has no access to old ideas.

The plan was always this direction. It just doesn't know otherwise.

Database migrations

The Worry:
Data loss

"Did the migration change what the data means?
Can I recover it if something went wrong?"

Level 0. One small mistake: unrecoverable data loss.

Universe: Result handling

1. Archive table — all rows preserved before migration runs

2. Bidirectional remapping — deterministic verification before execution

3. Extracted library — AI writes definition only; library executes

Deterministic pre  →  AI creative decision  →  deterministic execution.
