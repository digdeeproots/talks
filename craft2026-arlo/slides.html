Safety Categories

Carefree Right action easy. Wrong action structurally hard.
Negative toil Frees nearby attention too.
Prevention Mistakes can't propagate past the originator.
Zero toil Safe in scope.
Deterministic Catches foreseen errors and categories.
Focused toil Watch for "test net gaps."
Probabilistic Sometimes caught; unpredictable gaps.
Constant, low-grade worryAccept errors that QA misses.
Vigilance Caught only when someone is paying attention.
Constant toil Watch every action.
Hope No mechanism. Errors propagate undetected.
Zero toil until something fails silently.

The recipe

Engineering carelessness

  1. Name the vigilance. What are you watching for?
  2. Build a guardian. An AI check for some of it.
  3. Read its output as signal, not verdict. Pass and fail both classify what was checked: caught, missed, or verified-safe.
  4. Lock one class in with a universe lever at a chosen safety level.
  5. Make a decision Choose whether to accept the agent's work or to reset and try again with your new safety lever..

Discovery during demos

Vigilance cost:
Decision inconsistency

"Does this contradict a decision from earlier in the session?
Are we building on conflicting assumptions?"

Universe: Memory

  • Delete completed stories entirely (don't mark done)
  • Rewrite all future stories to reflect the current direction
  • Clear agent context or rewrite session history

The AI cannot blend old ideas with new ones.
It has no access to old ideas.

This direction was always the plan. It doesn't know otherwise.

Name the experience

  • Lever Memory
  • Safety level 4 — Prevention. Conflicting decisions are structurally impossible.
  • Scope Cross-session decision inconsistency.
  • Slack No re-explaining context every session.

Database migrations

Vigilance cost:
Data loss

"Did the migration change what the data means?
Can I recover it if something went wrong?"

Universe: State control

1. Archive table — all rows preserved before migration runs

2. Bidirectional remapping — deterministic verification before execution

3. Extracted library — AI writes definition only; library executes

Deterministic pre  →  AI creative decision  →  deterministic execution.

Name the experience

  • Lever State control (Determinism sandwich)
  • Safety level 4 / 5 — Data loss is structurally impossible.
  • Scope Schema migrations against live data.
  • Slack Migrations stop being scary.
